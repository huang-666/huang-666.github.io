Redis

Redis 合适的应用场景

  • 缓存
  • 排行榜
  • 计算器
  • 分布式会话
  • 分布式锁
  • 社交网络
  • 最新列表
  • 消息系统

除了 Redis 你还知道哪些 NoSQL 数据库？

MongoDB、MemcacheDB、Cassandra、CouchDB、Hypertable、Leveldb。

Redis 有哪些高级功能

消息队列、自动过期删除、事务、数据持久化、分布式锁、附近的人、慢查询分析、Sentinel 和集群等多项功能。

Redis 几个比较主要的可执行文件？分别是？

可执行文件 作用
redis-server 启动 Redis
redis-cli Redis 命令行客户端
redis-benchmark Redis 基准测试工具
redis-check-aof Redis AOF 持久化文件检测和修复工具
redis-check-dump Redis RDB 持久化文件检测和修复工具
redis-sentinel 启动 Redis Sentinel

Redis 有几种数据类型

基本数据类型：字符串（String）、哈希（hash）、列表（List）、集合（Set）、有序集合（zset）。

还有 HyperLogLog、流、地理坐标等。

如何查看当前键是否存在？

exists key

如何删除数据？

del key

Redis 为什么这么快？

因为：

  • Redis 使用了单线程架构和I/O多路复用模型。
  • 纯内存访问。
  • 由于是单线程，所以避免了上下文切换带来的资源消耗。

Redis 字符串数据类型最大不能超过多少？

==512MB==

Redis 默认有多少个数据库？

==16个==

Redis 持久化的几种方式？

RDB、AOF、混合持久化。

RDB 持久化

RDB（Redis DataBase）持久化是把当前进程数据生成快照保存到硬盘的过程。

注意：RDB 是以二进制的方式写入磁盘的。

RDB 持久化的优点

  • RDB 是一个紧凑的二进制文件，代表 Redis 在某个时间点上的数据快照。
  • 适合于备份，全量复制的场景，对于灾难恢复非常有用。
  • Redis 加载 RDB 恢复数据的速度远快于 AOF 方式。

AOF 持久化

AOF（append only file）为了解决 RDB 不能实时持久化的问题， AOF 来搞定，它以独立的日志方式把每次的命令记录记录到 AOF 文件中。

请你说一下什么是缓存穿透、缓存击穿、缓存雪崩？还有它们的解决方案

==缓存穿透==：缓存穿透是指查询一个一定不存在的数据，由于缓存是不命中时需要从数据库查询，查不到数据则不写入缓存，这将导致这个不存在的数据每次请求都要到数据库去查询，进而给数据库带来压力。

缓存穿透解决方案：

  1. 查询返回的数据为空，仍把这个空结果进行缓存，但过期时间会比较短；

  2. 布隆过滤器：将所有可能存在的数据哈希到一个足够大的 bitmap 中，一个一定不存在的数据会被这个 bitmap 拦截掉，从而避免了对 DB 的查询。

==缓存击穿==：缓存击穿是指热点key在某个时间点过期的时候，而恰好在这个时间点对这个Key有大量的并发请求过来，从而大量的请求打到db。

缓存击穿解决方案：

  1. 使用互斥锁：当缓存失效时，不立即去 load db，先使用如 Redis 的 setnx 去设置一个互斥锁，当操作成功返回时再进行 load db 的操作并回设缓存，否则重试 get 缓存的方法。

  2. 永远不过期：物理不过期，但逻辑过期（后台异步线程去刷新）。

==缓存雪崩==：缓存雪崩是指缓存中数据大批量到过期时间，而查询数据量巨大，引起数据库压力过大甚至宕机。

缓存雪崩解决方案：

  • 将缓存失效时间分散开，比如可以在原有的失效时间基础上增加一个随机值，比如 1-5 分钟随机，这样每一个缓存的过期时间的重复率就会降低，就很难引发集体失效的事件。
